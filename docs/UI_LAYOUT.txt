COLOR GUIDE:
═════════════════════════════════════════════════════════════════════════

Background Colors:
  Lavender (#E6E6FA)    - Main background
  Darker Lavender       - Table headers, inactive account buttons
  Light Lavender        - Odd table rows
  White                 - Even table rows, modals

Button Colors:
  Purple (#9370DB)      - Primary action buttons, active account
  Red (#D9534F)         - Delete buttons, negative values
  Gray                  - Cancel buttons, inactive states

Text Colors:
  Dark Gray (#333)      - Main text
  Red                   - Negative balances, over-budget amounts
  Purple                - Section headers in modals

Visual States:
  PURPLE + BORDER       - Active account button
  RED TEXT              - Negative balance or over budget
  CHECKMARK (✓)         - Transaction is reconciled
  RED X BUTTON          - Delete action


RESPONSIVE BEHAVIOR:
═════════════════════════════════════════════════════════════════════════

Desktop (>600px):
  • Full button labels visible
  • Table columns spacious
  • Modals centered

Mobile (<600px):
  • Buttons remain readable (min-width enforced)
  • Account buttons wrap to multiple rows if needed
  • Table scrolls horizontally if needed
  • Touch-friendly tap targets (22px minimum)
  • Modals fill 90% of screen width


WORKFLOW DIAGRAMS:
═════════════════════════════════════════════════════════════════════════

Daily Transaction Entry:
  1. Check active account button (purple highlight)
  2. Click [Add]
  3. Fill form (category auto-suggests from budget)
  4. Click [Save Transaction]
  5. Balance updates automatically

Monthly Reconciliation:
  1. Download bank CSV
  2. Click [CSV]
  3. Select "Auto Reconciliation" mode
  4. Click [Import CSV File]
  5. Choose downloaded file
  6. Review preview (15 new found)
  7. Check "Mark as reconciled" if desired
  8. Click [Confirm Import]
  9. Transactions matched and added

Budget Check:
  1. Click [Bdgt]
  2. Review spending by category
  3. Note any red amounts (over budget)
  4. Close modal
  5. Adjust spending as needed

Account Switching:
  1. Click desired account button (top row)
  2. Button turns purple
  3. Transactions update to show that account
  4. Add/edit/view transactions for that account


KEY INTERACTION PATTERNS:
═════════════════════════════════════════════════════════════════════════

✓ All modals close via:
  - [×] button (top right)
  - Click outside modal
  - Cancel/close buttons

✓ All destructive actions require confirmation:
  - Delete transaction
  - Delete account  
  - Delete budget category
  - Purge transactions
  - Import over existing data

✓ Visual feedback immediate:
  - Button highlights when clicked
  - Active account always visible
  - Balance recalculates instantly
  - Budget updates in real-time

✓ Error prevention:
  - Required fields enforced
  - Duplicate imports blocked
  - Default account protected
  - Fuzzy matching tolerant
